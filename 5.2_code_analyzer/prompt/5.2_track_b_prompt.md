## 0. 이 문서의 역할

Track B는 Track A 실행 전에 선택적으로 실행하는 정적 구조 추출이다.

```text
Track B
  → output/5.2/<slug>/structure_<ts>_focused.json 또는 structure_<ts>_full.json
  → Track A Phase 0이 같은 경로에서 자동 사용
```

대형 코드에서는 Track B를 먼저 실행하면 Track A의 토큰/탐색 비용이 줄어든다.

## 1. v0.2 고정 규칙

### 1.1 canonical path

Track B는 structure를 반드시 아래 경로에 저장한다.

```text
output/5.2/<slug>/
```

아래 경로는 사용하지 않는다.

```text
%USERPROFILE%\artifacts\code_analyzer\<slug>\
artifacts/5.2/<slug>/                            # v0.2 경로. v0.3에서 output/5.2/로 이동
artifacts\code_analyzer\<slug>\
```

### 1.2 structure 파일명 규칙

```text
focused structure: output/5.2/<slug>/structure_<YYYYMMDD_HHMM_KST>_focused.json
full structure:    output/5.2/<slug>/structure_<YYYYMMDD_HHMM_KST>_full.json
```

기본 산출물은 focused structure다. full structure는 300KB 이하일 때만 자동 인계 대상으로 삼는다. 300KB 초과 full structure는 사용자가 명시한 경우에만 Track A가 읽을 수 있다.

### 1.3 extraction_mode enum

Track B는 실제 사용한 추출 방식을 아래 enum 중 하나로 기록한다.

```text
extraction_mode: git | bash | powershell | internal_search | mixed
```

- `git`: git ls-files / git grep 중심
- `bash`: bash find/grep/xargs 중심
- `powershell`: Get-ChildItem / Select-String 중심
- `internal_search`: Claude Code 내부 파일 검색 중심
- `mixed`: 위 방식을 혼합

### 1.4 progress cursor

완료 진행줄:

```text

```

## 4. 추출 순서

```text
Step 1. slug 확정
  - 사용자가 target slug를 주면 최우선 사용
  - 없으면 block/API 이름에서 자동 생성
  - canonical path = output/5.2/<slug>/

Step 2. 환경 확인
  - git 사용 가능 여부
  - bash 사용 가능 여부
  - powershell 사용 가능 여부
  - ctags 사용 가능 여부
  - 실제 extraction_mode 확정

Step 3. 파일 목록 + LOC
  - 제외 폴더 적용
  - .c .cpp .h .hpp 중심
  - 파일 수와 total_loc 계산

Step 4. 후보 파일 축소
  - block/API 이름으로 후보 파일 먼저 찾기
  - 후보 파일 주변 include/callee 중심으로 1-hop, 필요 시 2-hop 확장
  - 전체 코드루트 무차별 body read 금지

Step 5. 함수 시그니처 추출
  - ctags 가능 시 ctags 사용
  - 없으면 grep/Select-String/internal_search fallback
  - raw symbol table 전체 덤프 금지

Step 6. IPC REQ 지점 찾기
  - 패턴별 최대 200줄까지만 1차 수집
  - caller 함수 + file + line 추출
  - 각 후보에 안정적인 id를 부여한다. 예: REQ001

Step 7. IPC CNF handler 후보 찾기
  - 단일 handler로 단정하지 않는다
  - `ipc_cnf_handlers[]` 후보 배열로 기록
  - 후보가 여러 개면 status=CANDIDATE, confidence 기록
  - 각 후보에 안정적인 id를 부여한다. 예: CNF001

Step 8. domainType / stackId / RAT 분기 위치 찾기
  - domain_branches[]에 curated 기록

Step 9. call_edges[] 합성
  - edge 본문이 아니라 id 기반 재참조가 가능해야 한다.
  - 각 edge에 `id`를 부여한다. 예: EDGE001
  - Phase 1이 procedure_runtime_index에 edge id만 가져갈 수 있게 한다.

Step 10. structure.json 합성
  - schema/structure.schema.md 준수
  - raw AST 금지
  - 전체 symbol table 덤프 금지
  - 크기 제한 적용

Step 11. 저장
  - focused면 output/5.2/<slug>/structure_<YYYYMMDD_HHMM_KST>_focused.json
  - full이면 output/5.2/<slug>/structure_<YYYYMMDD_HHMM_KST>_full.json
  - overwrite 금지
  - NEXT_STEP_5.2.md 갱신
```

## 5. PowerShell fallback 예시

bash 명령이 실패하면 PowerShell을 사용한다.

```powershell
$root = "<코드루트>"
$exclude = @("\test\", "\third_party\", "\build\")
$files = Get-ChildItem -Path $root -Recurse -File -Include *.c,*.cpp,*.h,*.hpp |
  Where-Object {
    $p = $_.FullName
    -not ($exclude | Where-Object { $p -like "*$_*" })
  }

$files | ForEach-Object {
  $lines = (Get-Content $_.FullName -ErrorAction SilentlyContinue | Measure-Object -Line).Lines
  [PSCustomObject]@{ File = $_.FullName; LOC = $lines }
}

Select-String -Path $files.FullName -Pattern "HAL_|_REQ|SendMsg|PostMsg" -CaseSensitive:$false
Select-String -Path $files.FullName -Pattern "_CNF|CnfHandler|MsgDispatch" -CaseSensitive:$false
```

위 예시는 참고용이다. 실제 실행 시에는 코드루트/제외폴더/성능을 고려해 필요한 범위만 수행한다.

## 6. structure 크기 제한

```text
300KB 이하: full 또는 focused 모두 가능. 자동 탐색 대상 가능
300KB~1MB: modules는 요약, target 관련 call_edges만 유지. focused 저장 권장
1MB 초과: focused structure 생성 필수
```

focused structure는 block/API 관련 subgraph만 포함한다.

## 8. 검증 기준

```text
- structure.json 저장 위치가 output/5.2/<slug>/ 이다.
- focused structure는 structure_<ts>_focused.json 이름을 사용한다.
- full structure는 structure_<ts>_full.json 이름을 사용한다.
- extraction_mode가 enum 중 하나다.
- raw AST가 없다.
- IPC REQ 후보가 ipc_req_sites[]에 들어간다.
- CNF 후보는 ipc_cnf_handlers[] 배열이다.
- req/cnf/edge 후보는 id를 가진다.
- 후보가 2개 이상이어도 실패하지 않고 CANDIDATE로 기록한다.
- grep/검색 결과는 패턴별 200줄 상한을 지킨다.
- structure가 1MB 초과 시 focused structure로 줄인다.
- NEXT_STEP_5.2.md가 canonical path 기준으로 갱신된다.
```
